Scene objects

Dragon Tool is a kind of a constructor. Working with it you create objects, adjust their properties and combine them. Objects are also called 'nodes' or 'tree nodes'. Node to which another node is attached is called 'parent' of attached node and an attached node is called 'child' node.

'Palette' node is a parent of a 'stone' node. 'Stone' node is a child of a 'palette' node.

The value of object property is set in the Property Editor.

Property Types

Each property can be one of a following types:
  • integer number (int). Example: 1, -4, 0, 3.
  • numbers with floating point (float). Example: 1.5, 0.3, 0.6.
  • numbers with high preceision (double). Example: 0.000002.
  • String. Example: bullet, status_bar.
  • boolean values (boolean). Example: true, false.
  • Objects. Some objects need references to other objects (for example, an IndexedTileMap needs a reference to a palette). This reference is defined in the following format:

    |node1|node2|referenceNode

    where referenceNode is the reference, and node2 is a parent of referenceNode, node1 is a parent of a node2.

    The path to the 'panelTile' node is '|panel|panelTile'.

    Some properties can be only read, but not set (for example, the width of the image). Every object has at least two properties: name and visible. Name of the object should consist of numbers and english letters. (It's possible to give an non-english letters in object names, but such names wouldn't be exported to mobile dragon.) Visible property defines whether or not Dragon Tool should draw this object and it's children.

    Dragon Tool supports following types of objects:

    Sprite

    Sprite is just a bitmap (or a picture) that is loaded into the program. It takes a picture from the png file and draws it on screen. When png file changes then sprite image updates automaticaly. Sprite has following set of properties:
    path (string)Relative path to the bitmap file. For examle: sprites/bullet.png. All png files should be stored inside the 'sprites' project folder or it's subfolders.
    width (integer, read only)Width of the image.
    height (integer, read only)Height of the image.
    showMatte (boolean)When this option is on all sprites in the scene show display it's transparent color as magenta.

    Transform

    Transform object defines location, rotation, scale, transparency and Z order of the attached objects. It has following properties:
    translateX (double)X coord of the object.
    translateY (double)Y coord of the object.
    ZOrder (integer)Z order of the transform. Objects with higher value of Z order will be painted over objects with lower values.
    rotation (double)Rotation angle. [POSITIVE and NEGATIVE angles].
    displayPivot (boolean)Check to display a pivot point. Pivot point is a point around which object is rotated.
    scaleX (double)Scale of the object on x axis (1 = 100% = no scale).
    scaleX (double)Scale of the object on y axis.
    opacity (float)Opacity of the object. Can vary from 0 (transparent) to 1 (opaque).

    Dummy

    Dummy is a simliest object in the programm. It has only two properties: name and visible. It usually used just for grouping a set of objects or as a container for custom properties.

    Grid

    Grid object is used to adjust a grid color/snapping. It's created every time you make a new project. Most part of it's properties is the same as a Transform properties. But some of them have slightly different meaning.
    showHomeGrid (boolean)Check it to display a small home grid.
    gridWidth (double)Size (or step) of the grid (x axis).
    gridHeight (double)Size (or step) of the grid (y axis).
    checkerOpacity (float)Opacity of the checker texture. Doesn't affect on the small home grid. (if you wish to hide it - uncheck the visible property).

    Polygon

    Polygon is a shape that is defined by a set of Points which should be attached to this object. Polygon is used for defining a custom tile map cell shape.

    Point

    Points define a shape of a polygon.

    Square Cell

    Cell is an object that is used together with tile maps.
    size (integer)Size of the cell (in pixels).

    Isometric Cell

    size (integer)Size of the cell (in pixels).
    aspectRatio (float)How distorted the cell is.

    Cell

    It is consists of three objects: a polygon, which defines it's shape and two points. This points affect the layout of the cells in the map. First point defines a location of the next cell in the row, second - location of the next cell in the column. There is an expample in the Quick Start section and a Working with IndexedTileMap tutorial that shows a role of a cell.

    TileMap

    This object (as says it's name) is a tile map. Size of the map is defined by rowCount and columnCount properties. Also you can turn on showCells property to display cells of the map. The cell property stores a path to a cell which is used in current tile map.

    IndexedTileMap

    The best analogue of an IndexedTileMap is a indexed image (for examle a GIF bitmap). Such image has a color palette and a data, which store only indexes of colors in this color palette. The same is with IndexedTileMap. It has a set of objects as a palette and a data which refer to idexes of this objects. See an expample in the Quick Start section and a tutorial Working with IndexedTilMap.

    IndexedTileMap properties:
    palette (object)Palette node which is used in this map. This node is defined in the Tree Path format.

    Time

    Time object is created every time you make a new project. It has a property frameNum which is a frame number for all animations in the scene. When you press Play button on the player panel frameNum increases with 30 frames per second.

    Animation

    Animation object has the same properties as a Transform object. The difference is that Animation object can have only Frame objects as it's children. Also it can display only one frame (the index of this frame is defined by a Time object).

    Frame

    Frame objects should be attached only to the animation objects. Their purpose is to define delay during wich this frame will be shown. See Working with Animation tutorial to see Frame objects in action.

    TilesExporter

    TilesExporter object created only once when you choose "DragonTool -> Export Res" command. In this object you can adjust output settings of the resource files.
    border (integer)Defines empty space between sprites in the output image (in pixels).
    imageSize (integer)Size of the output image (in pixels).
    outDir (string)Output directory where resource files are written.
